# Plataforma de Registro - Sistema de Gestión Universitaria

Este proyecto es una réplica funcional de un sistema de gestión universitaria, inspirado en los procesos administrativos de la Universidad Nacional Autónoma de Honduras (UNAH). La aplicación web está diseñada para manejar procesos clave como la admisión y matrícula de estudiantes, además de incluir portales personalizados para personal administrativo, docentes, jefes de departamento y coordinadores. Aunque no es el sistema oficial de la UNAH, este proyecto fue desarrollado como una solución educativa para abordar un problema de gran magnitud, poniendo en práctica conocimientos en diseño, arquitecturas de software y metodologías ágiles como Scrum.

## Mi rol en el proyecto

Como desarrollador full-stack y diseñador UX/UI, tuve un papel integral en la creación y evolución de esta réplica. Mis responsabilidades incluyeron:

### Diseño de interfaz de usuario

Utilicé **Figma** para crear mockups y prototipos de la aplicación, enfocándome en diseñar una interfaz intuitiva, visualmente atractiva y fácil de usar. Trabajé en la disposición de menús, botones, flujos de navegación y la organización de la información para garantizar una experiencia de usuario fluida y eficiente.

### Desarrollo frontend

Implementé las vistas y el comportamiento de la aplicación utilizando tecnologías como **JavaScript Vanilla**, **Bootstrap** y **HTML**. Mis tareas incluyeron:

- **Componentes interactivos**: Desarrollé formularios dinámicos, tablas de datos y menús de navegación para facilitar la interacción del usuario.
- **Integración con el backend**: Trabajé en la conexión entre el frontend y el backend, asegurando que los datos se mostraran y gestionaran correctamente.

### Desarrollo backend

Implementé la lógica del servidor utilizando **PHP**, enfocándome en funcionalidades específicas requeridas, la interacción con la base de datos y la creación de endpoints para gestionar los procesos de admisión y matrícula. Mis responsabilidades incluyeron:

- **Gestión de datos**: Diseñé y optimicé consultas SQL para gestionar la información de estudiantes, docentes y administradores.
- **Funcionalidades requeridas**: Trabajé en funcionalidades como la generación de un archivo CSV para exportar datos y la lectura de uno para insertarlos en la base de datos.

### Colaboración en metodologías ágiles

Participé activamente en la planificación y ejecución del proyecto utilizando **Scrum**, lo que permitió un desarrollo organizado y eficiente.

## Importancia del proyecto

Este proyecto es de gran relevancia porque, aunque es una réplica, aborda un problema real y complejo: la gestión de procesos universitarios en una institución de gran escala como la UNAH. Desarrollar esta aplicación nos permitió poner en práctica habilidades técnicas (diseño, programación, manejo de bases de datos) y metodologías ágiles como Scrum, lo que fortaleció nuestra capacidad para trabajar en equipo y gestionar proyectos de gran envergadura. Además, representa un avance hacia la digitalización de servicios universitarios, mejorando la eficiencia y la experiencia de los usuarios.

## Impacto en los usuarios

La réplica del Registro UNAH ofrece una solución que puede beneficiar a los usuarios de varias maneras:

- **Simplificación de procesos**: Facilita la gestión de admisiones y matrículas, reduciendo el tiempo y esfuerzo requeridos tanto para estudiantes como para personal administrativo.
- **Acceso centralizado**: Proporciona un sistema unificado donde estudiantes, maestros y administradores pueden gestionar sus tareas de manera eficiente.
- **Interfaz intuitiva**: Una interfaz bien diseñada y fácil de usar mejora la experiencia del usuario y reduce la curva de aprendizaje.
- **Transparencia y organización**: Los usuarios pueden acceder a información actualizada y gestionar sus procesos académicos de manera más transparente y organizada.

## Tecnologías utilizadas

- **Frontend**: JavaScript, Bootstrap, HTML, CSS.
- **Backend**: PHP.
- **Base de datos**: MySQL.
- **Diseño de interfaz**: Figma.
